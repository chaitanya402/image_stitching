Architecture

Video Generation Pipeline

End-to-end system for creating 9:16 carousel promotion videos from multiple product images using AI-generated SDXL banners, LLM-driven copy, and OpenCV compositing.

1 System Overview

🎯 Purpose

Takes N product images + a plain-text promotional description, generates two distinct AI banner backgrounds (top header + bottom footer) via SDXL, composites them with LLM-written copy, and encodes everything into a smooth-transition carousel MP4 optimised for social media (Instagram Reels / TikTok).

⚡ Key Design Decisions

  • Banners generated once per run — reused on every frame
  • Product images shown clean — no per-image AI editing
  • Slide transition moves only the product zone; banners stay pinned
  • LLM copy-gen has lightweight regex fallback (no hard dependency)
  • All AI inference is remote — no GPU required locally

🔌 External Dependencies

  • HF Router — router.huggingface.co routes SDXL images
  • InferenceClient — huggingface_hub manages auth + routing
  • HF Chat API — /v1/chat/completions for LLM copy (optional)
  • OpenCV — video encoding (mp4v codec)
  • Pillow — text overlays, badge compositing

📐 Output Spec

  • Resolution: 1080 × 1920 (9:16 portrait)
  • Codec: mp4v  /  Container: .mp4
  • Frame rate: 30 fps (configurable)
  • Hold time: 4 s per image (configurable)
  • Transition: 25 frames smoothstep slide-left
2 Pipeline Steps
1

Description Parsing DescriptionBasedIconAgent BannerContentAgent

Two agents operate on the plain-text description in parallel:

DescriptionBasedIconAgent uses regex to extract the discount percentage, classify product type (motorcycle / fashion / electronics / sport), and select a colour scheme + template.

BannerContentAgent calls a HuggingFace LLM (Mistral-7B → Qwen2.5-1.5B → SmolLM2-1.7B) via /v1/chat/completions on the HF Router to produce structured promotional copy. Falls back to intelligent phrase extraction if all LLM calls fail.

Output: headline, tagline, offer_points[3], badge_label Output: discount_percent, product_type, color_scheme
↳

LLM fallback chain: Mistral-7B → Qwen2.5-1.5B → SmolLM2-1.7B → regex extraction

2

Image Loading OpenCV cv2.imread

Each input path is loaded as a BGR NumPy array via cv2.imread(). Invalid paths are warned and skipped. At least one valid image is required to proceed.

Output: List[np.ndarray BGR]
3

AI Banner Generation — Top RemoteInferenceGenerator SDXL

ImageGeneratorFactory.create(backend) instantiates RemoteInferenceGenerator with stabilityai/stable-diffusion-xl-base-1.0. The prompt emphasises a dark navy-to-black gradient, thin gold geometric border lines, carbon-fibre corner texture, and a completely empty centre so the PIL text layer has a clean canvas. Output is 1080 × 345 px PIL Image.

Prompt style: "dark navy gradient, gold border lines, empty centre" Output: PIL Image (1080 × banner_height)
↳

Fallback: programmatic dark gradient via NumPy if SDXL call fails

3b

AI Banner Generation — Bottom RemoteInferenceGenerator SDXL

Second independent SDXL call with a distinct prompt: deep charcoal-to-black gradient, thin gold accent lines along top edge only, dark leather / carbon-fibre texture. Same size as top banner. Both calls share the same generator instance.

Prompt style: "charcoal gradient, gold top-edge lines, leather texture" Output: PIL Image (1080 × bottom_height)
4

PIL Compositing build_banner_frame build_bottom_banner_frame draw_discount_badge

Text and badges are composited onto the AI-generated backgrounds via Pillow:

Top banner — headline text (Arial Bold, gold) with semi-transparent rounded backing; gold circular badge geometry straddles the banner / product-zone boundary (badge_cx, badge_cy returned for later use).

Bottom banner — 2px gold separator line at top edge; offer bullets distributed vertically with auto font-size; each bullet wrapped in a dark semi-transparent rounded pill.

Output: banner_pil (top), bottom_banner_pil, badge params (cx, cy, r)
5

Frame Compositing compose_frame

Per frame, the product image is resize-cropped to fill the middle zone (height − banner_height − bottom_height = 1230 px), placed on a black NumPy canvas, converted to PIL, then both banner PILs are .paste()d at their fixed positions. The gold badge is drawn last (so it overlaps the zone boundary). Returned as OpenCV BGR NumPy array.

Output: np.ndarray BGR (1080 × 1920 × 3)
6

Video Encoding cv2.VideoWriter mp4v codec

For each image: fps × duration identical frames are written (static hold). Between images: 25 transition frames are generated using a smoothstep ease function — the product zone slides left while both banners are re-pasted at their fixed screen positions on every blended frame (so they never move). The full sequence repeats num_loops times.

Transition easing: smoothstep t = 3t²−2t³ Output: .mp4 file on disk
↳

Total frames ≈ (fps×duration + 25) × N_images × loops

7 Configuration

📄 config/config.json

{
  "huggingface": {
    "api_token": "hf_YOUR_TOKEN_HERE"
  },
  "generation": {
    "default_backend": "sdxl-remote",
    "inference_steps": 30,
    "guidance_scale":  3.0
  }
}

🔑 Token Resolution Order

  • 1. config/config.json  (primary)
  • 2. HF_TOKEN environment variable
  • 3. Raises ValueError if neither found
⚠ Security: config/config.json is gitignored. Copy config.json.example → config.json and paste your real token. Never commit the real token.

🏗 Available Backends

  • sdxl-remote — Free, fast ⭐ SDXL via HF Router
  • flux-remote — FLUX.1-dev via HF Router (requires credits)
  • hf-inference — HF Inference API free tier (optional dep)
Recommended: Use sdxl-remote — no GPU, no local model download, free tier, generates in ~10-20 s per banner.
8 CLI Reference
# Single run — explicit image paths
$ python generate_carousel_with_banners.py \
    "C:/input/jacket.jpg" "C:/input/helmet.jpg" \
    --description "flat 20% off on motorcycle gear, customization available" \
    --backend     sdxl-remote \
    --output      "C:/output/carousel.mp4"

# Process all images in the input folder
$ python generate_carousel_with_banners.py all \
    --description "summer sale 30% off on all sportswear"

# Custom resolution / timing
$ python generate_carousel_with_banners.py \
    "img1.jpg" "img2.jpg" "img3.jpg" \
    --description "buy 2 get 1 free on all electronics" \
    --width  1080  --height 1920 \
    --fps    30    --duration 5  --loops 3 \
    --steps  40    --backend  flux-remote
First-time setup: Run setup.ps1 (Windows) or setup.sh (Linux/macOS) to create the .venv, install dependencies, and be prompted to configure your HF token.
